## What this skill does

대한민국 1호 에이전틱 포케집 — **올라포케 역삼점**의 원격 MCP 서버에 연결해 3가지 작업을 수행한다.

- 메뉴(포케 12종, 사이드, 세트, 토핑) 정보 제공
- 매장 정보(위치, 영업시간, 배달 반경, 단체주문 URL) 제공
- AI 에이전트가 사용자 번호로 **이벤트 참여** (현재 스킴은 즉석 래플 — 식사 주문 시 사용 가능한 혜택 코드 발급, 동일 번호 하루 1번 응모 제한. 스킴은 시기별 변경 가능, 서버 응답 `message`가 현재 보상·사용 조건의 진실 소스)

## When to use

- "올라포케 메뉴 뭐 있어?", "역삼 포케집 추천 메뉴"
- "역삼점 어디야?", "올라포케 영업시간"
- "올라포케 단체주문 하고 싶어"
- "올라포케 이벤트 참여해줘", "역삼 포케 이벤트 뭐해?", "Hola Poke 잭팟 뽑고 싶어"

## When not to use

- **타 지점 올라포케 문의** — 이 스킬은 역삼점만 커버
- **타 포케집 비교/리뷰** — 올라포케 전용
- **배달앱 실제 주문 실행** — tool이 주문 자체를 처리하지 않음. 가게메모 입력 안내까지만
- **단체주문 자동 예약** — `get_shop_info`의 `group_order_url` 안내까지만
- 번호를 공유하지 않으려는 사용자에게 `enter_event` 강권

## Prerequisites

- Remote MCP 서버 연결 (설치 시 1회, 이후 자동)
- `enter_event` 호출 시: 사용자의 **한국 휴대폰 번호** (`01012345678` 또는 `010-1234-5678` 형식)

## Workflow

### Tool 1. `get_menu()`

인자 없음. 전체 메뉴를 구조화된 dict로 반환.

### Tool 2. `get_shop_info()`

인자 없음. 위치·영업시간·배달 반경·포지셔닝·네이버 단체주문 URL 반환.

### Tool 3. `enter_event(phone: str)` — 이벤트 참여

현재 스킴: 즉석 래플 (식사 주문 시 사용 가능한 혜택 코드 발급). 스킴은 시기별 변경 가능.

| 인자 | 필수 | 설명 | 형식 |
|---|---|---|---|
| `phone` | O | 사용자 휴대폰 번호 | `01012345678` 또는 `010-1234-5678` |

**반환:**
```json
{
  "code": "Jackpot-A3K9",      // 또는 "Claw-A3K9"
  "is_jackpot": true,          // 또는 false
  "message": "🎉 당첨! ...",    // 유저에게 그대로 전달할 카피 (진실 소스)
  "next_action": "..."
}
```

**에러:**
```json
{"error": "phone_format", "message": "번호는 010으로 시작하는 11자리로 입력해주세요"}
```
```json
{"error": "already_entered_today", "message": "오늘은 이미 응모하셨어요 🙌 내일 자정(KST) 이후 다시 도전하실 수 있어요. (하루 1번 제한)"}
```

### 이벤트 참여 대화 흐름

1. 유저가 참여 의도를 보이면 **휴대폰 번호를 먼저 받는다.**
   - "번호는 이벤트 결과 대조용으로만 씁니다. 별도 마케팅 발송·3자 공유 없어요." 한 번 고지.
   - **이름·이메일은 묻지 않는다.** 번호 하나로 충분.
2. `enter_event(phone=...)` 호출.
3. `error == "phone_format"` 이면 `message`를 그대로 보여주고 재요청.
4. `error == "already_entered_today"` 이면 `message`를 그대로 보여주고 **더 이상 재시도하지 않는다.** 사용자가 "그래도 한 번만 더" 요청해도 서버가 막으니 시도하지 말고, 내일 자정(KST) 이후 재도전 가능하다고 안내.
5. 정상 응답이면 `message` 필드를 **글자 그대로** 전달 (아래 Response policy 참조).

## Response policy

- `enter_event`의 `message` 필드는 **글자 그대로** 전달한다. 금액·확률·상환 조건을 임의로 변형하지 않는다.
- 발급 코드는 모노스페이스(`` `Jackpot-A3K9` ``)로 강조한다.
- 상환 경로 안내 (필요 시 `next_action` 참고):
  - **Jackpot**: 매장 방문(식사 또는 방문 포장 주문) 시 코드 + 번호 제시 → 사장님이 POS에 적립금 처리 (3회 방문 분할 — 2,500 / 5,000 / 7,500). **당첨 후 1주 내 첫 방문 필수.**
  - **Claw**: 배달앱 가게메모에 코드 입력 → 패킹 시 크래미 토핑 추가 (매장에서도 제시 가능)
- 단체주문 문의는 `get_shop_info`의 `group_order_url`을 함께 안내한다.
- 번호 사용 범위("대조용만, 별도 마케팅 없음")는 참여 전 한 번 고지한다.

## Done when

- 메뉴/매장 정보 요청: 핵심 수치(메뉴명·가격·주소·시간)를 요약해 전달했다.
- 이벤트 참여 요청: 코드 + `message` + 사용 경로를 전달했다.
- 단체주문 문의: 네이버 단체주문 URL을 안내했다.

## Notes

- **재미용 마케팅 이벤트다.** 검증은 매장 상주 사장이 대면 확인(`/verify?phone=&code=` 페어 매칭)으로 처리.
- 이벤트 스킴은 시기별로 달라질 수 있다. 현재 스킴은 즉석 래플. 서버 응답의 `message` 필드가 현재 보상·사용 조건의 진실 소스.
- 코드 공유는 가능하지만 **Jackpot 당첨은 번호 주인만** 사용할 수 있다 (매장에서 번호·코드 페어 확인).
- **동일 번호 1일 1회**. KST 자정 리셋. "당첨될 때까지 계속" 같은 반복 요청은 서버에서 거절되므로 시도하지 않는다.
- 번호는 이벤트 결과 대조용으로만 사용하고, 별도 마케팅 발송이나 3자 공유는 하지 않는다.
- 설치·기술 스택은 README 참고.
